Manual Book E-BUDIHARTANTO
Platform Agensi Teknologi & LMS

Panduan lengkap penggunaan aplikasi Budi Hartanto Platform — mencakup akses login (email & kata sandi), peran pengguna, dashboard admin, sistem pembelajaran (LMS), pelacak proyek klien, struktur database, dan panduan instalasi.

1. Pendahuluan

E-BUDIHARTANTO (Budi Hartanto Platform) adalah sebuah aplikasi web berbasis Laravel 13 (PHP 8.3) yang menggabungkan dua layanan utama dalam satu ekosistem:

  • Tech Agency — jasa pengembangan perangkat lunak (aplikasi web/sistem dan aplikasi mobile) beserta pelacakan proyek untuk klien.
  • LMS (Learning Management System) — sistem kursus pemrograman online dengan video pembelajaran, modul materi, dan pelacakan progres untuk siswa.

Aplikasi ini dibangun dengan antarmuka modern (Tailwind CSS v4, Alpine.js, AOS animations) dan mendukung tiga jenis pengguna berbeda: Admin Student Client.

Manual book ini disusun berdasarkan kode sumber aktual aplikasi (routes/web.php, controller, model, view, dan DatabaseSeeder) sehingga seluruh informasi akurat dan dapat diandalkan.

2. Fitur Utama Platform

Portofolio

Galeri proyek nyata (web/sistem & mobile) dengan filter kategori, badge teknologi, dan deskripsi production-grade.

Layanan Agensi

Paket jasa pengembangan dengan harga transparan, daftar fitur, dan tombol konsultasi.

LMS Kursus

Kelas pemrograman dengan pemutar video, daftar materi, dan progres belajar.

Pelacak Proyek

Status proyek real-time (Pending → Designing → Developing → Completed) untuk klien.

Panel Admin

Kelola order, ubah status proyek & pembayaran, pantau metrik platform.

Autentikasi

4. Instalasi & Setup

Aplikasi dijalankan di atas MAMP (MySQL port 8889) dengan PHP 8.3. Berikut langkah instalasi lengkap:

4.1 Prasyarat

  • PHP ≥ 8.3, Composer, Node.js & npm
  • MAMP (MySQL di 127.0.0.1:8889, user root / pass root)
  • Database MySQL bernama e_budihartanto

4.2 Langkah Instalasi

1
Install dependensi PHP & JS:
composer install
npm install
2
Salin file environment & buat APP_KEY:
cp .env.example .env
php artisan key:generate
3
Pastikan konfigurasi database di .env sesuai:
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=8889
DB_DATABASE=e_budihartanto
DB_USERNAME=root
DB_PASSWORD=root
4
Jalankan migrasi & isi data awal (termasuk akun login di atas):
php artisan migrate --seed
5
Build aset frontend & jalankan server:
npm run build
php artisan serve --port=8000
Akses http://localhost:8000. (Saat menggunakan MAMP, arahkan ke document root public/.)
Jika tampilan terlihat tanpa styling, jalankan npm run dev atau npm run build karena aset di-load melalui Vite manifest.

6. Panduan Halaman Publik

Halaman beranda (/) dapat diakses tanpa login dan terdiri dari beberapa section:

  • Hero — pengantar platform dengan tombol "Lihat Portofolio" & "Mulai Belajar".
  • Portofolio (#portfolio) — grid proyek dengan filter Semua / Web-Sistem / Mobile (Alpine.js).
  • Layanan (#services) — paket agensi (Web, Mobile, WordPress) beserta harga & fitur.
  • Kursus LMS (#courses) — daftar kelas; tombol "Akses Kelas" (jika sudah login) atau "Daftar Kelas".
  • Kontak (#contact) — form konsultasi (simulasi alert).

7. Autentikasi (Masuk & Daftar)

7.1 Masuk (Login)

  1. Buka /login (atau klik "Masuk" di navbar).
  2. Masukkan Email dan Kata Sandi (lihat bagian 3.1).
  3. Centang "Ingat Saya" (opsional) untuk sesi tahan lama.
  4. Klik Masuk Sekarang. Sistem akan mengarahkan ke dashboard sesuai peran.
Jika kredensial salah, akan muncul pesan: "Kredensial yang diberikan tidak cocok dengan catatan kami." Pastikan email & kata sandi tepat.

7.2 Daftar (Register)

  1. Buka /register → isi Nama, Email, Pilih Peran, Kata Sandi (min. 8 karakter), Konfirmasi.
  2. Peran hanya bisa student atau client.
  3. Setelah berhasil, otomatis login & diarahkan ke dashboard masing-masing.

7.3 Keluar (Logout)

Klik Logout pada navbar. Sesi diakhiri dan Anda dikembalikan ke beranda.

8. Panel Admin (Dosen / Pengajar)

Akses: /admin/dashboard (hanya role admin). Menampilkan metrik dan manajemen proyek.

8.1 Kartu Metrik

Total Siswa

Jumlah user dengan role student.

Total Klien

Jumlah user dengan role client.

Pendapatan (Paid)

Total total_amount dari order berstatus paid.

8.2 Tab: Pelacakan Proyek Klien

Tabel order berisi Nomor Invoice, Klien, Layanan, Total Biaya, dan pemilih status. Admin dapat mengubah:

  • Status Proyek: pending → designing → developing → completed
  • Status Pembayaran: pending, paid, cancelled

Klik Update untuk menyimpan perubahan (mengirim POST ke /admin/orders/{id}/status).

8.3 Tab: Daftar Kursus LMS

Grid semua kursus beserta jumlah pelajaran dan harga — referensi materi pembelajaran siswa.

9. Dashboard Siswa (LMS)

Akses: /student/dashboard (hanya role student). Menampilkan grid kursus yang tersedia.

9.1 Mengakses Kelas

  1. Klik kartu kursus → tombol Masuk Kelas.
  2. Diarahkan ke /student/courses/{id} (player video + materi).

9.2 Halaman Belajar (Course View)

  • Pemutar Video HTML5 di sisi kiri dengan judul & isi materi aktif.
  • Daftar Materi di sisi kanan — klik untuk berganti pelajaran (Alpine.js, tanpa reload).
  • Setiap materi menampilkan nomor urut, judul, durasi (menit), dan konten.
Progres ditandai dengan materi yang sedang aktif (highlight indigo). Video otomatis dimuat ulang saat mengganti materi.

13. FAQ & Troubleshooting

T: Saya lupa kata sandi, bagaimana reset?

Fitur reset melalui email belum aktif (mailer disetel log). Untuk keperluan development, ubah kata sandi lewat tinker: User::where('email','admin@budihartanto.com')->update(['password'=>Hash::make('password')]);

T: Masuk dengan benar tapi dikembalikan ke beranda?

Pastikan session & cookie berjalan (driver database — tabel sessions harus ada via migrasi). Jalankan php artisan migrate bila tabel belum dibuat.

T: Role salah saat coba akses dashboard?

Middleware role akan mengalihkan ke dashboard peran Anda sendiri. Pastikan login dengan akun sesuai peran (lihat 3.1).

T: Tampilan berantakan / tanpa style?

Jalankan npm run build (atau npm run dev). Aset dimuat via Vite manifest dari resources/.

T: Bagaimana menambah akun Admin baru?

Gunakan php artisan tinker: User::create(['name'=>'...','email'=>'...','password'=>Hash::make('password'),'role'=>'admin']);

T: Email & kata sandi default untuk demo?

admin@budihartanto.com / student@example.com / client@example.com — semua dengan kata sandi password (lihat bagian 3.1).
